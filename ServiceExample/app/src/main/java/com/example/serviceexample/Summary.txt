*   Service

    - 사용자에게 노출되지 않도록 백그라운드에서 지속적으로 처리해야 하는 경우에 사용.
    - 서비스는 기본 스레드를 사용하므로 메인 스레드와 서비스 중 하나에만 처리를 집중해야 함.
    - 기본 스레드와는 상관없이 다중적인 작업처리가 가능한 스레드와는 차이점이 있음.

    @ 서비스의 두 가지 종류 (서비스가 시작하는 방식)
    - Started Service
        - startService() 메소드로 시작.
        - Activity와는 무관하게 독립적으로 동작. (Activity가 종료되어도 계속 실행됨.)

        - 서비스를 시작하는 버튼 + 서비스를 중지하기 위한 버튼 총 2개로 started service 테스트

    - Bind Service
        - bindService() 메소드로 시작.
        - Activity와 특정 데이터를 주고받을 필요가 있을 경우 사용. (Activity가 종료되면 함께 종료)
